# Versionamento do Design System (inicial)

## Convenção
- **Patch**: correções internas sem mudança de API (ex.: ajuste de CSS/estilo).
- **Minor**: novas features compatíveis (nova variante/prop opcional).
- **Major**: breaking changes (remoção/renomeação de props, mudança de comportamento).

## Regras
- Breaking change deve incluir:
  - Motivo
  - Guia de migração
  - Atualização no Storybook
