# Getting Started

Bem-vindo ao Design System da **Academia Maestro**! 

Este Storybook documenta todos os componentes, tokens e hooks reutilizáveis utilizados no projeto. Use esta documentação para explorar, entender e implementar os componentes do design system de forma consistente.

## Sobre o Design System

O Design System da Academia Maestro foi criado para garantir consistência visual e de experiência em toda a aplicação. Todos os componentes seguem princípios de acessibilidade, usabilidade e manutenibilidade.

## Como usar este Storybook

- **Explorar componentes**: Navegue pelas diferentes categorias de componentes na sidebar
- **Ver exemplos**: Cada componente possui múltiplas variações e exemplos de uso
- **Copiar código**: Use os exemplos como referência para implementar os componentes
- **Entender props**: Todas as propriedades estão documentadas com exemplos interativos

## Como importar

### Tokens
```typescript
import { tokens } from '../design-system/tokens'
import { colors, typography, spacing } from '../design-system/tokens'
```

### Componentes
```typescript
import { Button, Input } from '../design-system/components'
// ou individualmente
import { Button } from '../design-system/components/Button'
```

### Hooks
```typescript
import { useModal, ToastProvider } from '../design-system/hooks'
```

## Convenções

- **Composição**: Preferir composição de componentes base em vez de criar novos do zero
- **Tokens**: Sempre usar tokens (cores, espaçamento, tipografia) em vez de valores hardcoded
- **Acessibilidade**: Todos os componentes seguem padrões de acessibilidade por padrão
  - Navegação por teclado funcional
  - Foco visível e semântico
  - Atributos ARIA quando necessário
- **Documentação**: Manter componentes bem documentados com exemplos claros

## Estrutura do Design System

- **Tokens**: Cores, tipografia, espaçamento - a base visual do sistema
- **Componentes**: Elementos reutilizáveis agrupados por categoria
- **Hooks**: Funcionalidades reutilizáveis para lógica de componentes

## Próximos passos

1. Explore a página de [Tokens](./?path=/story/design-system-tokens--docs) para entender as variáveis de design
2. Navegue pelos componentes por categoria (Formulários, Navegação, Layout, etc.)
3. Use os exemplos interativos para testar diferentes configurações dos componentes
